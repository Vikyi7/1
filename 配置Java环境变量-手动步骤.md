# 手动配置 Java 环境变量

如果自动脚本无法运行（需要管理员权限），请按以下步骤手动配置：

---

## 步骤 1：找到 Java 安装路径

1. 打开 **文件资源管理器**
2. 导航到：`C:\Program Files\Java\`
3. 找到 `jdk-25` 或 `jdk-25.0.1` 文件夹
4. **复制完整路径**（例如：`C:\Program Files\Java\jdk-25`）

**如果找不到**：
- 按 `Win + R`，输入 `appwiz.cpl`，按回车
- 找到 "Java(TM) SE Development Kit 25.0.1"
- 右键 → **"更改"** → 查看安装路径

---

## 步骤 2：打开环境变量设置

1. 按 `Win + X`
2. 选择 **"系统"**
3. 点击 **"高级系统设置"**（在右侧）
4. 点击 **"环境变量"** 按钮

---

## 步骤 3：添加 JAVA_HOME

1. 在 **"系统变量"** 区域（下方），点击 **"新建"**
2. **变量名**：`JAVA_HOME`
3. **变量值**：粘贴刚才复制的 Java 安装路径
   - 例如：`C:\Program Files\Java\jdk-25`
   - **注意**：路径中**不要**包含 `bin` 文件夹
4. 点击 **"确定"**

---

## 步骤 4：添加到 PATH

1. 在 **"系统变量"** 区域，找到 **"Path"** 变量
2. 选中 **"Path"**，点击 **"编辑"**
3. 点击 **"新建"**
4. 输入：`%JAVA_HOME%\bin`
5. 点击 **"确定"**

---

## 步骤 5：保存并验证

1. 点击 **"确定"** → **"确定"** → **"确定"**
2. **关闭所有已打开的命令行窗口**
3. **重新打开命令行**（按 `Win + R`，输入 `cmd`，按回车）
4. 输入以下命令验证：
   ```cmd
   java -version
   ```
5. 如果显示版本信息，说明配置成功 ✅

---

## 如果还是不行

1. **重启电脑**（环境变量更改有时需要重启才能生效）
2. 重启后再次运行 `java -version` 验证

---

## 验证成功示例

运行 `java -version` 后，应该看到类似：

```
java version "25.0.1" 2024-xx-xx
Java(TM) SE Runtime Environment (build 25.0.1+xx-xx)
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+xx-xx, mixed mode, sharing)
```
